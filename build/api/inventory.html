Inventory API

The Inventory API provides endpoints for managing products, stock levels, and inventory movements.

Products​

List Products​

GET /api/inventory/products
Authorization: Bearer {token}

Response

{
  "items": [
    {
      "id": "string",
      "name": "string",
      "sku": "string",
      "category": "string",
      "unit": "string",
      "stock_level": "number",
      "reorder_point": "number"
    }
  ],
  "total": "number",
  "page": "number",
  "size": "number"
}

Create Product​

POST /api/inventory/products
Authorization: Bearer {token}
Content-Type: application/json

{
  "name": "string",
  "sku": "string",
  "category": "string",
  "unit": "string",
  "reorder_point": "number"
}

Stock Movements​

Record Movement​

POST /api/inventory/movements
Authorization: Bearer {token}
Content-Type: application/json

{
  "product_id": "string",
  "type": "RECEIPT|ISSUE|TRANSFER|ADJUSTMENT",
  "quantity": "number",
  "reference": "string",
  "notes": "string"
}

Get Movement History​

GET /api/inventory/movements
Authorization: Bearer {token}

Stock Levels​

Get Current Stock​

GET /api/inventory/stock-levels
Authorization: Bearer {token}

Update Stock Level​

PUT /api/inventory/stock-levels/{product_id}
Authorization: Bearer {token}
Content-Type: application/json

{
  "quantity": "number",
  "reason": "string"
}

Error Responses​

Product Not Found​

{
  "detail": "Product not found"
}

Invalid Movement​

{
  "detail": "Invalid movement type"
}

Insufficient Stock​

{
  "detail": "Insufficient stock for operation"
}

Best Practices​

  1. Always validate stock levels before operations
  2. Use appropriate movement types
  3. Include meaningful references
  4. Handle errors gracefully
  5. Monitor stock level changes

Sales API

The Sales API provides endpoints for managing sales orders, customers, and invoices.

Sales Orders​

List Orders​

GET /api/sales/orders
Authorization: Bearer {token}

Response

{
  "items": [
    {
      "id": "string",
      "customer_id": "string",
      "order_date": "string",
      "status": "string",
      "total_amount": "number",
      "items": [
        {
          "product_id": "string",
          "quantity": "number",
          "unit_price": "number",
          "total": "number"
        }
      ]
    }
  ],
  "total": "number",
  "page": "number",
  "size": "number"
}

Create Order​

POST /api/sales/orders
Authorization: Bearer {token}
Content-Type: application/json

{
  "customer_id": "string",
  "items": [
    {
      "product_id": "string",
      "quantity": "number",
      "unit_price": "number"
    }
  ]
}

Customers​

List Customers​

GET /api/sales/customers
Authorization: Bearer {token}

Create Customer​

POST /api/sales/customers
Authorization: Bearer {token}
Content-Type: application/json

{
  "name": "string",
  "email": "string",
  "phone": "string",
  "address": {
    "street": "string",
    "city": "string",
    "state": "string",
    "postal_code": "string",
    "country": "string"
  }
}

Invoices​

Generate Invoice​

POST /api/sales/invoices
Authorization: Bearer {token}
Content-Type: application/json

{
  "order_id": "string"
}

Get Invoice​

GET /api/sales/invoices/{invoice_id}
Authorization: Bearer {token}

Error Responses​

Order Not Found​

{
  "detail": "Order not found"
}

Customer Not Found​

{
  "detail": "Customer not found"
}

Invalid Order​

{
  "detail": "Invalid order data"
}

Best Practices​

  1. Validate order data before submission
  2. Check stock availability
  3. Handle customer information securely
  4. Process payments safely
  5. Keep order history updated
